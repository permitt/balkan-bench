# frontend

The BalkanBench landing page and public leaderboard. React 19 + Vite 8 + `react-router-dom`. Deployed to Vercel from this directory (repo `rootDirectory = frontend`).

## Routes

| Path           | Purpose |
|----------------|---------|
| `/`            | Landing page: live release hero, email capture, status card. |
| `/leaderboard` | Renders `public/leaderboards/superglue-serbian/benchmark_results.json`. Sortable by any task column or Avg. |
| `/about`       | Methodology summary, language + scope, sponsor block, links to the spec. |
| `/submit`      | Step-by-step submission flow + links to the GitHub issue templates. |

## Local dev

```bash
cd frontend
npm install
npm run dev
```

Visit `http://localhost:5173/`, `/leaderboard`, `/about`, `/submit`.

## Build

```bash
npm run build
npm run preview
```

`npm run build` must stay green; CI's `ci.yml` does not currently exercise the frontend but the GitHub repo has Vercel previews on every PR.

## How the leaderboard JSON stays fresh

`public/leaderboards/superglue-serbian/benchmark_results.json` is the source of truth. It is regenerated by the `balkanbench leaderboard export` CLI from the official result artifacts committed under `eval/results/official/` (once Plan 7 lands).

Today the file is seeded with the v1.0 launch rows from the user's verified prior-pipeline numbers. When the new pipeline regenerates results, copy the output back into that path and commit.

```bash
cd ../eval
source .venv/bin/activate
balkanbench leaderboard export \
  --benchmark superglue \
  --language sr \
  --results-dir results/official \
  --out ../frontend/public/leaderboards/superglue-serbian/benchmark_results.json
```

## Deploy to Vercel

1. Connect the GitHub repo at [vercel.com/new](https://vercel.com/new).
2. Set **Root Directory** to `frontend`.
3. Framework preset: **Vite** (picked up automatically).
4. The committed `vercel.json` sets the SPA rewrite so deep links resolve (`/leaderboard` reloads instead of 404'ing).
